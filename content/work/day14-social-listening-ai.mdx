## The Problem

1. Brand mentions go unnoticed for 72+ hours — viral complaints damage reputation before anyone responds
2. Manual monitoring of Reddit, news, and review sites wastes 30–60 mins/day
3. Enterprise tools (Brandwatch, Mention.com) cost $300–$1,000+/month for features most teams use 20% of
4. Generic alerts say "someone mentioned you" — no context, no urgency, no action guidance
5. No team awareness system — the right person finds out last when something blows up

## The Solution

A scheduled n8n workflow that:
- Pulls up to 50 brand mentions from Reddit + NewsAPI every 6 hours (100% free sources)
- Normalizes both sources into a single unified mention schema
- Runs each mention through GPT-4o-mini: 10 intelligence fields including sentiment, urgency, intent, emotion, priority score (1–10), and a pre-written suggested response
- Logs a 20-column intelligence database to Google Sheets automatically
- Fires a rich Slack alert for every high-urgency mention — with the draft response already written
- Secondary Slack "response queue" for any mention flagged as needing a reply

## How It Works

Node-by-node flow:

- **Schedule Trigger** — runs every 6 hours
- **Set** — single config block (brand, API keys, Sheets ID, Slack channel)
- **HTTP Request (×2)** — Reddit API + NewsAPI
- **Code (×2)** — Normalizes both sources into unified schema
- **Merge** — combines both streams
- **SplitInBatches** — processes one mention at a time
- **HTTP Request (OpenAI)** — GPT-4o-mini: sentiment, intent, urgency, emotion, priority, suggested response
- **Code** — safe JSON parser with fallback on error
- **Google Sheets** — 20-column intelligence row per mention
- **IF (×2)** — routes high urgency and response-needed mentions
- **Slack (×2)** — urgent alert + response queue channel
- **No Op** — silent pass for monitor/ignore actions
