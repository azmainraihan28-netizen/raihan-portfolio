## The Problem

1. SaaS companies lose 40–60% of new signups in the first 7 days — mostly because onboarding is generic and feels like it was built for someone else
2. Customer success teams manually write the same onboarding emails for every new customer — 30–60 minutes of repetitive work per person
3. Day 3 check-ins and Day 7 follow-ups are "planned" but never actually sent — too many customers, not enough time
4. Every customer gets the same onboarding flow regardless of role, industry, or use case — destroying activation rates
5. Tools like Intercom, Customer.io, and Appcues cost $200–$1,000/month for flows that still aren't truly AI-personalized

## The Solution

A webhook-triggered multi-agent n8n workflow that runs 3 AI agents in parallel the moment a customer signs up:
- **Agent 1 (Welcome Writer):** Reads customer name, role, industry, use case → writes a fully personalized welcome email with their specific 3-step action plan
- **Agent 2 (Onboarding Strategist):** Builds a 6-step role-specific checklist with time estimates, quick win, 14-day success milestone, and recommended features
- **Agent 3 (Health Monitor):** Pre-writes a Day 3 check-in email + a Day 5 re-engagement email — both queued automatically via Wait nodes before welcome is even sent
- All 3 agents run in parallel — full onboarding sequence generated in under 8 seconds
- Logs 15 customer intelligence fields to Google Sheets
- Fires Slack alert with health risk summary for ops team

## How It Works

Node-by-node flow:

- **Webhook** — POST /customer-onboarding
- **Set** — Normalizes 13 input fields (name, email, company, plan, industry, role, use_case, product, slack config, sheet ID)
- **IF** — Validates email not empty → 400 error on missing
- **HTTP Request (OpenAI ×3)** — Agent 1, Agent 2, Agent 3 run in parallel (fan-out)
- **Code** — Merges all 3 agent outputs, builds enriched welcome email HTML, builds Slack message
- **Gmail (×3)** — Welcome (instant), Day 3 check-in, Day 5 re-engagement
- **Wait (×2)** — 3-day delay, then 2-day delay (Day 5 total)
- **Google Sheets** — 15-column customer onboarding log
- **IF** — Routes Slack-enabled vs. silent
- **Slack** — Rich alert: customer card, quick win assigned, health risks, email queue status
- **No Op** — Silent pass when Slack disabled
- **Respond to Webhook (×2)** — Returns success JSON or 400 validation error
