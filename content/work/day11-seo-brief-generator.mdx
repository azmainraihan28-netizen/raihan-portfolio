## The Problem

1. Manual brief creation wastes 3–5 hours per article — analyzing 10 competitor pages, PAA, headings, gaps every time
2. Writers start with vague briefs ("write 1,500 words on X") and produce content that never ranks
3. SERP data goes stale fast — manual research doesn't keep up with ranking changes
4. SEO brief tools (Surfer SEO, Clearscope, MarketMuse) cost $100–$500/month — overkill for freelancers
5. Teams publish content 80% identical to competitors, missing the gaps that could rank them #1

## The Solution

A webhook-triggered workflow that:
- Accepts a keyword via POST request and hits SerpAPI for live top-10 Google SERP data
- Extracts competitor titles, snippets, People Also Ask questions, and related searches
- Sends structured SERP data to GPT-4o-mini which generates a full content brief as JSON
- Brief includes: 3 title options, meta description, full H1→H3 outline, FAQ section, secondary keywords, content gaps, CTA, competitor insights
- Logs every brief to Google Sheets and emails the HTML-formatted version directly to the writer
- Complete brief generated in ~25 seconds from keyword input

## How It Works

Node-by-node flow:

- **Webhook** — Receives keyword + config (POST)
- **Set** — Normalizes inputs and sets defaults
- **IF** — Guards against empty keyword → 400 error
- **HTTP Request** — SerpAPI call for live SERP data
- **Code** — Extracts top results, PAA questions, related searches
- **OpenAI (gpt-4o-mini)** — Generates full structured content brief as JSON
- **Code** — Parses AI JSON, builds HTML email version
- **Google Sheets** — Logs keyword + brief metadata (12 columns)
- **IF** — Routes to Gmail send or silent skip
- **Gmail** — Sends formatted brief to writer's email
- **NoOp** — Silent skip when no email provided
- **Respond to Webhook (×2)** — Returns brief JSON or 400 error
