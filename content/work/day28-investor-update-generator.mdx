## The Problem

1. Founders spend 2–4 hours per investor update manually writing, formatting, and proofreading — every single month
2. Metrics live in 4 different tools (Stripe, Mixpanel, QuickBooks, spreadsheets) — there is no single narrative view, making updates inconsistent
3. Inconsistent or skipped updates damage investor confidence — silence is worse than bad numbers, yet founders skip updates when busy
4. Generic investor update templates feel impersonal — investors disengage from obvious copy-paste emails with no analysis
5. No historical record — founders can't see their own progress arc or reference past updates at board meetings without hunting through email threads

## The Solution

A webhook-triggered n8n workflow that:
- Accepts raw business metrics via POST: MRR, ARR, customers, churn, burn rate, runway, cash on hand, plus founder commentary (wins, challenges, asks)
- Sends all data to GPT-4o-mini which returns: a full 750–1,000 word HTML investor update, subject line, health score (1–10), health label, MRR/customer growth %, key insight, risk flag, and next raise signal
- Emails every investor on the list automatically — handles comma-separated addresses, sends individually
- Logs 23 fields to Google Sheets — building a permanent investor relations database
- Posts a rich Slack block digest with health score, key insight, risk flag, and executive summary
- Returns a structured JSON response for front-end or Notion/Airtable integration
- Cost: ~$0.002 per update (GPT-4o-mini)

## How It Works

Node-by-node flow:

- **Webhook** — POST /investor-update
- **Set** — Normalizes 22 input fields with safe defaults
- **IF (Validate)** — Guards against missing company_name or zero MRR → 400 error
- **HTTP Request (OpenAI)** — GPT-4o-mini with JSON mode: generates full investor update package (10 fields)
- **Code (Parse)** — Safely parses AI JSON, merges with input context, computes token cost
- **Google Sheets** — Appends 23-column investor update log row
- **IF (Has Emails?)** — Routes to email send path or silent skip
- **Code (Split Emails)** — Splits comma-separated investor list into individual items
- **Gmail** — Sends personalized HTML update to each investor
- **IF (Slack Enabled?)** — Routes to Slack digest or silent skip
- **Slack** — Rich block digest: health score, growth %, key insight, risk flag, summary, cost
- **No Op (×2)** — Silent passes for no-email and no-Slack paths
- **Respond to Webhook (×2)** — Returns full JSON result (200) or validation error (400)
