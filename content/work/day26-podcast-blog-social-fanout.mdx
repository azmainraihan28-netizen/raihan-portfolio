## The Problem

1. Podcasters spend 60+ minutes recording and 0 minutes distributing — the ratio is backwards
2. Manual repurposing (blog + tweets + LinkedIn + email + TikTok) takes 4–8 hours per episode
3. Tool stack (Descript, Buffer, Jasper, ChatGPT) costs $100+/month and still requires manual effort
4. Most AI tools hallucinate podcast summaries — they need the actual transcript, not just a topic
5. Publishing cadence breaks across platforms — no unified momentum when episode drops

## The Solution

A webhook-triggered n8n workflow that:
- Accepts a podcast episode URL + optional transcript via POST
- If no transcript: downloads audio → sends to OpenAI Whisper for multilingual transcription
- If transcript provided: skips Whisper entirely (faster, cheaper)
- Sends transcript to GPT-4o-mini to generate 6 content assets: SEO blog post (HTML), 10-tweet thread, LinkedIn post, 3 email paragraphs, 5 TikTok hooks, 3 key quotes
- Auto-publishes blog post to WordPress via REST API (draft or live — configurable)
- Logs 16 fields to Google Sheets content database
- Posts a rich Slack digest with all assets + WP link + API cost (optional)
- Returns full JSON with all content for front-end or CMS integration
- Cost: ~$0.004 per episode (Whisper + GPT-4o-mini)

## How It Works

Node-by-node flow:

- **Webhook** — POST /podcast-fanout
- **Set** — Normalizes 16 input fields with safe defaults
- **IF (Validate)** — Guards against missing episode_title or both url+transcript → 400 error
- **IF (Has Transcript?)** — Routes to transcript path or Whisper transcription path
- **Code (Use Provided Transcript)** — Passes existing transcript to merge point
- **HTTP Request (Download Audio)** — Downloads binary audio file from URL
- **HTTP Request (Whisper)** — Transcribes audio via OpenAI Whisper multipart form upload
- **Code (Extract Whisper Result)** — Validates and extracts Whisper transcript
- **Merge** — Unifies both transcript paths into single execution path
- **HTTP Request (OpenAI GPT-4o-mini)** — Generates all 6 content formats as structured JSON
- **Code (Parse AI Content)** — Safely parses AI output, computes token cost, builds full record
- **HTTP Request (WordPress)** — Creates blog post via WP REST API with app password auth
- **Google Sheets** — Logs 16-column content production record
- **IF (Slack Enabled?)** — Routes to Slack notification or silent skip
- **Slack** — Rich block digest: blog link, tweet count, LinkedIn preview, TikTok hooks, cost
- **No Op** — Silent pass when Slack is disabled
- **Respond to Webhook (×2)** — Returns full content JSON (200) or validation error (400)
