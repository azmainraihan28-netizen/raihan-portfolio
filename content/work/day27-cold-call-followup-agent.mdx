## The Problem

1. 80%+ of sales calls end with zero follow-up — reps intend to follow up but volume makes it impossible
2. Manual follow-up emails after 50 daily calls take 10–15 minutes each — reps skip them under pressure
3. Follow-up arrives 3–6 hours after the call — the prospect has already moved on or forgotten the conversation
4. No record of call context — reps show up to the next touchpoint blind, having to re-ask the same questions
5. Hot prospects go cold in under 2 hours without an immediate re-engagement signal post-call

## The Solution

A Twilio-triggered n8n workflow that:
- Triggers automatically when Twilio fires a call status webhook (CallStatus = completed)
- Filters out calls under 60 seconds (voicemails, wrong numbers, cold hang-ups)
- Sends call metadata + rep notes to GPT-4o-mini to generate a unique personalized SMS + email + lead score
- AI outputs: SMS message (≤160 chars), email subject, HTML email body, lead temperature (hot/warm/cold), urgency score (1–10), next step recommendation, follow-up day
- Sends SMS via Twilio within 90 seconds of call ending
- Sends personalized HTML follow-up email via Gmail in parallel
- Logs 17 fields to Google Sheets: full call intelligence record
- Fires Slack block alert to manager channel with lead card and next action

## How It Works

Node-by-node flow:

- **Webhook** — Receives Twilio CallStatus POST callback
- **Set** — Normalizes 19 input fields (Twilio fields + custom metadata)
- **IF** — Filters short/incomplete calls (\< 60 seconds OR status ≠ completed)
- **HTTP Request (OpenAI)** — GPT-4o-mini: 8-field follow-up package as JSON
- **Code** — Safely parses AI JSON with full fallback template
- **HTTP Request (Twilio)** — Sends SMS via Twilio Messages API (Basic Auth)
- **Gmail** — Sends personalized HTML follow-up email
- **Google Sheets** — Appends 17-column call + follow-up log
- **IF** — Routes Slack-enabled vs. silent
- **Slack** — Rich block manager alert with lead card, urgency, next step, SMS preview
- **No Op** — Silent pass when Slack disabled
- **Respond to Webhook (×2)** — Returns success JSON or skip JSON
