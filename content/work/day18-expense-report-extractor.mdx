## The Problem

1. Employees waste 20–30 minutes per expense manually entering data — multiplied across 50+ people this is days of lost productivity per month
2. Finance teams spend 5–10 hours/month reconciling reports with wrong categories, missing tax data, and duplicate entries
3. Tools like Expensify, Concur, and Ramp charge $10–$25 per user/month — expensive at scale, still not truly AI-powered
4. Paper and email receipts get lost or delayed, causing reimbursement friction and unhappy employees
5. No consistent categorization — different employees code the same expense differently, making reporting unreliable

## The Solution

A webhook-triggered n8n workflow that:
- Receives receipt text from any source (email forward, web form, mobile app, Zapier)
- Uses GPT-4o-mini to extract 16 structured fields: vendor, date, amount, tax, subtotal, category, subcategory, payment method, business purpose, billability, policy flag, confidence score
- Enforces a configurable budget limit — auto-flags OVER_LIMIT or NEEDS_REVIEW expenses
- Logs a 21-column row to Google Sheets for every submission
- Fires a rich Slack block message to the finance/manager channel for any policy violation
- Sends HTML confirmation email to employee — either auto-approval or pending review notice
- Returns structured JSON for front-end, ERP, or CRM integration
- Cost: ~$0.001 per receipt (GPT-4o-mini)

## How It Works

Node-by-node flow:

- **Webhook** — POST /expense-report
- **Set** — Normalizes 10 input fields with safe defaults (budget limit, currency, slack channel, sheet ID)
- **IF** — Guards against missing receipt_text or employee_email → 400 error
- **OpenAI (gpt-4o-mini)** — Extracts 16-field expense intelligence report as JSON
- **Code** — Safely parses AI JSON, applies budget limit check, builds final 21-field record
- **Google Sheets** — Appends 21-column expense log row
- **IF** — Routes policy-flagged vs. clean expenses
- **Slack** — Rich block message alert for manager/finance channel (approval required path)
- **Gmail (×2)** — Pending approval email OR auto-approval confirmation email
- **Respond to Webhook (×2)** — Returns success JSON (200) or validation error (400)
