## The Problem

1. Managers waste 15–30 minutes every morning chasing team members for status updates via DM or meeting
2. Async standups (forms, bots) collect data but do nothing intelligent with it — blockers get buried
3. Standup info lives and dies in Slack threads — no searchable history, no trend tracking, no health data
4. Blockers go unnoticed until sprint review or missed deadline — killing team velocity
5. Tools like Geekbot, Standuply, and Range cost $3–$9 per user/month with no real AI analysis built in

## The Solution

A webhook-triggered n8n workflow that:
- Receives standup submissions via POST from any form, Slack slash command, or custom app
- Uses GPT-4o-mini to structure the update, detect blockers, rate severity, estimate productivity, and read sentiment
- Posts a clean, formatted standup to the Slack team channel automatically
- Fires a private blocker alert to the manager channel if severity is medium or high
- Logs 16 intelligence fields to Google Sheets — building a long-term team health database

## How It Works

Node-by-node flow:

- **Webhook** — Receives POST request from team member
- **Set** — Normalizes 12 input fields with safe defaults
- **IF** — Validates required fields → returns 400 on missing
- **OpenAI (gpt-4o-mini)** — 10-field standup intelligence report (blockers, severity, risk, sentiment, productivity, Slack message, manager note)
- **Code** — Safe JSON parser with fallback template on parse error
- **Google Sheets** — Appends 16-column standup log row
- **Slack (×2)** — Team standup channel post + manager blocker alert
- **IF** — Routes blocker vs. clean standup
- **No Op** — Silent pass for clean standups
- **Respond to Webhook (×2)** — Returns 200 success JSON or 400 validation error
