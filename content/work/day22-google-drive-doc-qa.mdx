## The Problem

1. Teams waste 30–60 minutes/day searching Drive for information that already exists in documents
2. New employees have no intelligent way to query internal knowledge — they ask colleagues instead of reading docs
3. Sales reps can't instantly find product specs, pricing, or case studies during live customer calls
4. Enterprise search tools (Guru, Notion AI, Confluence AI) cost $300–$2,000/month — overkill for most teams
5. Answers given without citing sources are inconsistent — different team members say different things
6. Most teams think RAG requires engineers and vector databases — n8n makes it zero-infrastructure

## The Solution

A webhook-triggered n8n workflow that:
- Accepts a plain-English question via POST request
- Uses Google Drive's native full-text search API to find relevant documents (no vector DB needed)
- Exports content from Google Docs, Sheets, Slides, PDFs, and text files automatically
- Truncates and aggregates up to 5 documents into a structured GPT-4o-mini context prompt
- Returns a JSON answer with: full answer, source citations, confidence level, follow-up questions
- Logs every Q&A pair to Google Sheets for a searchable team knowledge history
- Optional Slack alert with the answer summary

## How It Works

Node-by-node flow:

- **Webhook** — POST /doc-qa with question, folder_id, max_docs, asked_by, sheet_id, slack_enabled
- **Set** — Normalizes all 8 input fields with safe defaults
- **IF** — Validates question not empty → 400 error on missing
- **HTTP Request (Drive Files API)** — Searches Google Drive with full-text query + optional folder scope
- **Code** — Parses Drive results, filters supported file types, returns N items for loop
- **HTTP Request (Drive Export)** — Exports Google Workspace files as text OR downloads PDFs
- **Code** — Re-attaches file metadata, cleans and truncates text to 3,500 chars per doc
- **Aggregate** — Collects all document excerpts into a single item
- **Code** — Builds structured RAG context prompt with document separators
- **HTTP Request (OpenAI)** — GPT-4o-mini with JSON mode: answer + sources + confidence + follow-ups
- **Code** — Safe JSON parser with fallback on parse error
- **Google Sheets** — Logs 12-column Q&A history (question, answer, sources, confidence, tokens, cost)
- **IF** — Routes Slack-enabled vs. silent
- **Slack** — Rich block message with full answer, sources, follow-up questions
- **No Op** — Silent pass when Slack disabled
- **Respond to Webhook (×2)** — Returns structured JSON answer (200) or validation error (400)
