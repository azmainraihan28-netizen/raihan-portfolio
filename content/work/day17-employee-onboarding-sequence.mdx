## The Problem

1. HR manually writes the same onboarding emails for every new hire — 45–90 minutes of repetitive work per person
2. Generic "Welcome to the team!" emails feel cold and impersonal — new hires disengage in their first week
3. Follow-up check-ins (Day 7, Day 30) are forgotten 60–70% of the time under busy ops schedules
4. Managers get zero structured briefing — new hires flounder because nobody told the manager what to do on Day 1
5. Platforms like BambooHR, Workday, and Monday cost $300–$2,000+/month for features n8n handles for $0.001 per hire

## The Solution

A webhook-triggered n8n workflow that:
- Accepts new hire data via POST (name, email, role, department, start date, manager info)
- Uses GPT-4o-mini to generate a full personalized onboarding package: 3 email bodies (Day 0/7/30), Slack announcement, manager briefing, 8-item checklist, buddy suggestion, 3 first-week goals
- Sends warm, personalized welcome email instantly on trigger
- Automatically sends Day 7 check-in and Day 30 milestone emails via n8n Wait nodes — no cron job needed
- Posts a Slack team announcement in the configured channel
- Emails the manager a structured first-week action briefing with checklist and goals
- Logs all 14 fields to Google Sheets for a permanent new hire database

## How It Works

Node-by-node flow:

- **Webhook** — POST /new-employee-onboarding
- **Set** — Normalizes 11 input fields with safe defaults
- **IF** — Guards against missing email/name → 400 error
- **OpenAI (gpt-4o-mini)** — Generates full personalized onboarding package as structured JSON
- **Code** — Safely parses AI JSON with full fallback template
- **Gmail (×3)** — Welcome email (Day 0), Check-in email (Day 7), Milestone email (Day 30)
- **Wait (×2)** — 7-day delay, then 23-day delay (total: Day 30)
- **Slack** — Team announcement for new hire
- **IF** — Routes to manager email or silent skip
- **Gmail** — Manager briefing with checklist + goals
- **No Op** — Silent pass when no manager email
- **Google Sheets** — 14-column new hire log
- **Respond to Webhook (×2)** — Returns success JSON or 400 error
