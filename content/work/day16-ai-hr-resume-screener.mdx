## The Problem

1. Manual resume screening takes 6–10 hours per open role — most of that time spent on obvious mismatches
2. Scoring is inconsistent — same resume gets a "Yes" from one recruiter, "No" from another
3. Top candidates rejected because of keyword mismatches with ATS, not actual skill gaps
4. High-volume job posts (200+ applications) are physically impossible to review thoroughly
5. ATS platforms (Greenhouse, Lever, Workable) charge $300–$2,000/month for screening features that still aren't truly AI-powered

## The Solution

A webhook-triggered n8n workflow that:
- Accepts resume text + job description via POST request
- Uses GPT-4o-mini to generate a 14-field screening report in ~3 seconds
- Scores candidates 1–10 with match %, strengths, gaps, red flags, culture fit, interview questions, salary estimate, and next action
- Logs an 18-column row to Google Sheets for every candidate
- Fires Slack alert + HTML email ONLY for top candidates (score ≥ 7) — zero noise for low scorers
- Returns structured JSON response for front-end or ATS integration
- Cost: ~$0.001 per resume (GPT-4o-mini)

## How It Works

Node-by-node flow:

- **Webhook** — POST /hr-resume-screener
- **Set** — Normalizes 8 input fields (name, email, resume, JD, position, company, hr_email, timestamp)
- **IF** — Guards against missing resume_text or job_description → 400 error
- **OpenAI (gpt-4o-mini)** — 14-field structured screening report as JSON
- **Code** — Safely parses AI JSON with full fallback on parse error
- **Google Sheets** — 18-column candidate screening database
- **IF** — Routes top candidates (score ≥ 7) vs. low scorers
- **Slack** — Rich top-candidate alert with full profile summary
- **Gmail** — Branded HTML email to HR with full screening report
- **No Op** — Silent pass for low-score candidates
- **Respond to Webhook (×2)** — Returns results JSON or 400 error
